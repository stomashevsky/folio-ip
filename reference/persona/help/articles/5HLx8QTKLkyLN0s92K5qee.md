# Database Verification (Mexico)

_The ability to use the pre-built Database Verification (MX) solution is available on select plans. Customers can contact your Customer Success Manager or [Customer Support](https://app.withpersona.com/dashboard/contact-us) for more information._

## Database Verification (MX) Features[](#database-verification-mx-features)

Persona's Database Verification product offers the ability to verify attributes of an individual against authoritative Mexican databases. This helps your business gain an understanding of users associated with Mexico. Using the out-of-the-box Mexico Database Verification, you can verify attributes like name, date of birth, address, and certain national identification numbers against known matches in authoritative databases.

Learn more about [Database Verification](./7dcdaIsMttmPUZ2ZelRbZD.md).

### Inputs[](#inputs)

Using the default set up for Mexico Database Verification, users are asked to provide their details through an Inquiry flow. There are 4 required inputs for Database Verification (MX):

1.  Name of the individual
2.  Birthdate
3.  National ID - we accept 2 types:
    1.  Clave Única de Registro de Población (CURP) - E.g. ABCD123456ABCDEF1
    2.  Registro Federal de Contibuyentes (RFC) - E.g. A1B2C2D4E5F6G
4.  Address details

### Outputs[](#outputs)

The Database Verification checks whether the information above presents a sufficient match in our Mexican databases. To learn how to understand Database Verification results, check out [this article](./7Lh9MkK1DUczNlAe4EB6oW.md).

## Configuration Options[](#configuration-options)

The default attributes and match configurations on the out-of-the-box Database Verification (MX) solution are recommended settings suitable for most businesses seeking a balance of high assurance and user conversion.

Advanced configurations such as type of authoritative database matched against, options for matching against issuing databases, additional attribute verification (phone number, email address, etc), and per attribute advanced match requirement configs are available on our Enterprise plan.

### Match Requirements[](#match-requirements)

The Database Verification configurations and checks specify what it means for a user’s information to “match” against a known database.

The match requirements rules for Database Verification (MX) can be viewed and adjusted under the **Identity Comparison** check when viewing your specific Verification Template in the dashboard. It is recommended to proceed with caution when deviating from the default match requirements for Database Verification (MX) as it can directly impact your user's ability to proceed through the Inquiry flow.
