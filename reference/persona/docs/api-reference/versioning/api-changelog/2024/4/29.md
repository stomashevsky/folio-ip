### Accounts & Transactions

-   ğŸƒ **Asynchronous redaction of related resources**: Previously, the [Account](../../../../accounts/redact-an-account.md) and [Transaction](../../../../transactions/redact-a-transaction.md) redaction APIs would redact the primary object (Account or Transaction) as well as any related resources (Inquiries, Verifications, etc.) synchronously. Now these APIs will redact the primary object synchronously and redact any related resources asynchronously. The `account.redacted` and `transaction.redacted` events will only be emitted after all objects are redacted. This should improve performance.
